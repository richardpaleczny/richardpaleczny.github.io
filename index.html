    What is the difference between ‘switch’ and ‘if/else’? When would you use one versus the other?

  • I think the difference is that "switch" statements are more for matching cases, similar to chained "if/else if" statements, albeit in an easier format to read. I would use "switch" statements if I were trying to match a large number of cases, so that the person reading the code would be more able to quickly process what's happening. General "if/else" statements have a large number of uses - in particular, in checking certain logical conditions or if there had to be nested "if" statements for more specific actions. I find it easier to write them out when there are not a lot of redundant checks that have to be made (in this regard, switch statements might look nicer). They're more useful at refining the logic to solve a problem. But I think "switch" and "if/else if" statements are probably interchangeable.

    • What is the difference between ‘while’ and ‘do while’ loops? Give an example of when you would use one versus another. What is the benefit of a ‘do while’ over a ‘while’?

  • A "while" loop will repeat its code block as long as its conditional statement is "true". Similarly, a "do while" loop will also continue to loop while its conditional is "true", but the caveat is that a "do while" loop also performs its code block exactly once before its conditional gets checked. This is the basis for using a "do while" loop - and perhaps that's because you might not be really sure whether to check the condition until something happens - e.g. prompting a user for information, getting their input, and based on that input, decide whether or not to keep running the loop - but you have to run the block at least once to prompt them in the first place, say. Alternatively, you could perform the same logic without the "do while" loop, but it's just easier to use the "do while" loop in this regard.
